103 ## 3. 更新ルール
104 - 追加したテストコードは、同時に本表へ追記します。
105 - `状態` は `Planned -> Executable -> Implemented` を基本遷移とし、実行環境未整備のみ `Deferred` を使用します。
106 - 受入ID（`AT-*`）は、範囲記法（`~`）を使わず単独IDで記載します。
107 - `AGENTS.md` 7.1 のPhase1必須範囲は `Deferred` を使用しません。
108 - Phase2対象は `備考` に `Phase2` を明記して追跡します。
