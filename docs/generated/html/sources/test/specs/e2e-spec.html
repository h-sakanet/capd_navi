29 ## 3. 共通実装ポリシー
30 - テストIDは `E2E-*` を使用し、`AT-*` と1:1または1:Nで紐付けます。
31 - セレクタは `getByRole` / `getByLabel` / `getByText` を優先します。
32 - 同名ボタン等で曖昧な箇所のみ `data-testid` を許可します。
33 - テストデータは deterministic fixture を使用し、ランダム依存を禁止します。
34 - 待機は `expect(...).toBeVisible()` 等の状態待機を使い、固定sleepを禁止します。
35 - fixture の基準ディレクトリは `/Users/sakanet/capd_navi/test/fixtures` に固定します。
36  
37 ### 3.1 状態定義
38 - `Planned`: ケース定義済み、テストコード未作成
39 - `Executable`: 現行実装/環境で実行可能
40 - `Implemented`: テストコード作成済み・定常実行対象
41 - `Deferred`: 依存I/Fや実行環境が未整備のため保留
42  
43 ### 3.2 実行ルール
44 - `Executable` / `Implemented` は `test:e2e` 実行対象とし、`skip/disable` を禁止します。
45 - `Deferred` は仕様とトレーサビリティで管理し、通常スイートには含めません。
46  
47 ### 3.3 初回実装必須ゲート（Deferred禁止）
48 初回実装では、以下のE2Eケースを必須対象とし `Deferred` を禁止します。
49 - `E2E-CSV-001`〜`E2E-CSV-004`
50 - `E2E-API-001`, `E2E-API-004`
51 - `E2E-API-002`
52 - `E2E-SYNC-001`〜`E2E-SYNC-006`
53 - `E2E-RECOVERY-001`〜`E2E-RECOVERY-003`
54 - `E2E-ALARM-001`〜`E2E-ALARM-004`
55  
56 運用:
57 - 着手時は `Planned` で管理し、依存I/F実装後に `Executable` 化します。
58 - 初回実装完了時点で、上記はすべて `Implemented` である必要があります。
59  
60 ### 3.4 Phase2対象（初回対象外）
61 以下はPhase2対象とし、初回実装では `Deferred` を許可します。
62 - `E2E-CSV-005`
63 - `E2E-PLAT-001`, `E2E-PLAT-002`, `E2E-SLEEP-001`
64 - `E2E-EXIT-001`〜`E2E-EXIT-007`
65 - `E2E-PHOTO-001`, `E2E-BACKUP-001`

158 ## 6. 実行順序（推奨）
159 1. 初回実装必須ゲート（3.3）の `Planned` ケースを依存I/F実装と合わせて `Executable` 化
160 2. `状態=Executable` の `E2E-FLOW-*` と `E2E-API-003` を実装
161 3. 初回実装必須ゲート（3.3）を `Implemented` まで完了
162 4. Phase2対象（3.4）はフェーズ開始時に `Deferred -> Planned -> Executable` へ変更して着手
