7 ## 1. 目的
8 CAPD v1 を「共通Web + Macネイティブシェル」で実装しつつ、データ基盤を `IndexedDB（端末正本） + Netlify Blobs（共有/バックアップ）` に固定します。

45 ## 3. 設計原則
46 - セッション進行ロジックはクライアント共通実装で一元化する
47 - データ正本は端末ローカル（IndexedDB）とする
48 - クラウドは共有・バックアップ用途に限定する
49 - 競合は同期時に LWW で内部自動解決する
50 - 同一端末内の進行中セッションは1件に制限する
51 - 同期データは非暗号化保存とし、HTTPS通信とURL管理で運用リスクを抑制する
52  
53 ## 4. 主要コンポーネント
54 - Web UI: Next.js（ホーム、セッション、記録一覧、設定）
55 - Local Domain Service: `SessionService`, `RecordService`, `DailyPlanService`
56 - Local Persistence: IndexedDB（sessions, records, daily plans, snapshots, outbox, sync state）
57 - Sync Service: 起動/復帰/完了/手動時に `push + pull`
58 - API: Netlify Functions（`POST /sync/pull`, `POST /sync/push`）
59 - Shared Storage: Netlify Blobs（`index.json`, `days/*.json`, `photos/*.jpg`）
60 - Backup: 日次スナップショット保持（30日）
61

87 ## 6. 非機能方針
88 - セキュリティ: アプリ内認証なし（公開URL運用）を継続し、HTTPS通信とURL管理ルールで補完
89 - 可用性: 通信断時もローカル継続、復帰後同期で最終整合
90 - 拡張性: `record_event` 追加を想定した列駆動設計
91 - 保守性: 同期契約（manifest/day bundle/outbox）を文書で固定
92  
93 ## 7. 開始時スナップショット契約
94 - 固定対象:
95 - `sourceProtocol`（`protocolId`, `protocolVersion`, `importedAt`）
96 - step定義本文（通し番号、`step_id/next_step_id`、フェーズ/状態、タイトル、表示/警告文、必須チェック、`timer/alarm/record` 指示）
97 - `assetManifest`（`sourceRelativePath` と `assetKey` 対応、`contentSha256`、`sizeBytes`）
98 - `snapshotHash`（正規化JSONの `sha256`）
99 - 保存方式:
100 - セッション開始時に `sessions` と `session_protocol_snapshots` を同一ローカルトランザクションで保存
101 - 失敗時は開始処理をロールバックし、エラーを表示
102 - 復元方式:
103 - セッション再開時は `session_protocol_snapshots` のみ参照
104 - 現行テンプレート版へのフォールバックは禁止
105 - 欠落またはハッシュ不整合は表示を停止し、復旧導線を提示
