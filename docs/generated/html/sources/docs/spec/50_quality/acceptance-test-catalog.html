3 ## 1. テスト方針
4 - 判定優先度は「安全制御」「記録完全性」「同期整合」「復旧性」です。
5 - 1件エラーで取り込み中止の原則を検証します。
6 - ローカル正本とクラウド共有の最終的整合が成立することを重点検証します。
7  
8 ## 1.1 実行参照
9 - 動線起点は `../10_journeys/journeys.md` と `../10_journeys/JRN-*.md` を使用します。
10 - 画面遷移と操作は `../20_screens/SCR-*.md` の `SCR-*` / `ACT-*` を使用します。
11 - 入力項目と表示順は `../20_screens/SCR-002-HOME-SETUP.md` と `../20_screens/SCR-007-SESSION-RECORD.md` の `FC-*` を使用します。

104 ## 3. 境界値テスト
105 - 必須チェック0件ステップで遷移可能か
106 - 最終ステップのみ `next_step_id` 空欄許容か
107 - セル内改行を含む `必須チェック` が正しく分割されるか
108 - 同一セッション内で複数 `alarm_id` が独立して再通知管理されるか
109 - 段階再通知（T+2分、T+5分以降3分間隔）がタイマーずれなく動作するか
110 - tombstone と upsert が同時到着した場合にLWWで一意決定できるか
111  
112 ## 4. 回帰テスト観点
113 - CSVパーサ更新で既存v3が読めなくならないこと
114 - 記録モーダル変更時に進行ブロック条件が崩れないこと
115 - 版管理変更時にスナップショット再現性が維持されること
116 - 同期ロジック変更時にLWW規則が崩れないこと
117 - API最小化方針（`sync/push`, `sync/pull` のみ）が維持されること
