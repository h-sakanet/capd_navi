1 # Storage Model Contract
2  
3 ## 1. 目的
4 - データモデルと保存先（IndexedDB / localStorage / Cloud）を明確化し、実装時の解釈ズレを防ぎます。
5 - FR-102（ローカル正本）、FR-091/FR-092（写真容量とバックアップ）の保存対象を固定します。
6  
7 ## 2. 正本ストレージ（IndexedDB）
8 - 正本は IndexedDB です。`localStorage` を正本として扱ってはいけません。
9 - オブジェクトストア（v1）:
10 - `protocol_packages`: `ProtocolPackage`（CSV取込済みテンプレート）
11 - `daily_procedure_plans`: `DailyProcedurePlan`（当日4スロット）
12 - `sessions`: `Session`
13 - `records`: `RecordEntity`
14 - `session_protocol_snapshots`: `SessionProtocolSnapshot`
15 - `outbox_mutations`: `OutboxMutation`
16 - `sync_state`: `SyncState`
17 - `alarm_dispatch_jobs`: `AlarmDispatchJob`
18 - `photo_meta`: `photoRefs` 相当メタ
19  
20 ### 2.1 `ProtocolPackage` 保存形（v1）
21 - `protocol_packages` の保存オブジェクトは、次の構造を必須とします。
22  
23 ```json
24 {
25 "meta": {
26 "formatVersion": 3,
27 "protocolId": "capd_onebag",
28 "protocolName": "ワンバッグ手技",
29 "protocolVersion": "v3.0.0",
30 "effectiveFromLocal": "2026-02-10T00:00:00+09:00"
31 },
32 "steps": [],
33 "source": {
34 "importMode": "mac_directory",
35 "basePath": "protocol_package"
36 },
37 "importedAt": "2026-02-09T13:00:00+09:00",
38 "validationReport": {
39 "errors": [],
40 "warnings": []
41 }
42 }
43 ```
44  
45 - `meta.effectiveFromLocal`: テンプレート有効開始日時（端末ローカル日時文字列）。
46 - `source.importMode`: v1では `mac_directory` 固定。
47 - `source.basePath`: 取り込みディレクトリ基準パス（`protocol.csv` 配置基準）。
48 - `validationReport`: 取り込み時の検証結果（`errors` / `warnings`）を保持。
49  
50 ## 3. Cloud共有/バックアップ（Netlify Blobs）
51 - `index.json`: `cloudRevision`, `dayRefs`, `photoRefs`, `tombstones`, `integrityHash`
52 - `days/{dateLocal}.json`: `DayBundle`
53 - `photos/{photoId}.jpg`: JPEGバイナリ
54 - FR-092 の「日次バックアップ」は、上記 `index.json + days/* + photos/*` の世代バックアップを指します。
55  
56 ## 4. localStorage（補助用途）
57 - 目的: 一時的なUI復元。正本ではありません。
58 - 現行キー（実装）:
59 - `capd-support:home:slots:v1`
60 - 値: `Array<ProcedureSlot | null>`（4スロット）
61 - 要素: `{ protocolId, protocolLabel, recommendedTime, status }`
62 - `capd-support:home:active-session:v1`
63 - 値: `{ sessionId, slotIndex, currentStepId, updatedAtIso }`
64 - この領域は将来的に IndexedDB へ統合可能な補助キャッシュとして扱います。
65  
66 ## 5. 保存ルール
67 - `Session.start` 時は `sessions` と `session_protocol_snapshots` を同一トランザクションで保存します。
68 - `Record` 更新時は `records` 更新と `outbox_mutations` 追記を同時に実行します。
69 - `exit_site_photo` 更新は `patch_path=payload.exit_site_photo` の部分パッチで保存します。
70 - `cloudState=missing` 判定時にローカル正本（IndexedDB）を削除してはいけません。
