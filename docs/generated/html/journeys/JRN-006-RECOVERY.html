JRN-006-RECOVERY 詳細

JRN-006-RECOVERY 復旧（DB消失/クラウド欠損）

対応FR

  • FR-087 - FR-087: IndexedDB消失検知時はクラウドからフルリストアを実行します。
  • FR-087A - FR-087A: POST /sync/pull が cloudState=missing を返した場合、クラウド欠損と判定します。
  • FR-087B - FR-087B: クラウド欠損判定時はローカルデータを正本として syncMode=full_reseed で全量再シードを実行し、ローカルデータは削除/初期化しません。
  • FR-087C - FR-087C: 全量再シード成功後は再度 POST /sync/pull を実行し、cloudState=ok と cloudRevision 更新を確認して同期完了とします。
  • FR-087D - FR-087D: 全量再シード失敗時はローカルデータを不変のまま保持し、lastSyncStatus=failed と再試行導線を表示します。
  • FR-088 - FR-088: 同期失敗時は直近同期状態（成功/失敗）を更新し、再試行導線を表示します。

対応AT

  • AT-RECOVERY-001 DB消失復元
  • AT-RECOVERY-002 クラウド欠損再シード
  • AT-RECOVERY-003 再シード失敗時保全

画面と操作

SCR

  • SCR-011-SYNC-STATUS 同期状態表示

ACT

  • ACT-SYNC-001 自動同期（起動/復帰/完了）
