JRN-005-SYNC 詳細

JRN-005-SYNC 同期と再試行

対応FR

  • FR-080 - FR-080: 同期は startup / resume / session_complete / manual の4契機で実行します。
  • FR-081 - FR-081: すべてのローカル更新は outbox に追記し、push成功時に消し込みます。
  • FR-082 - FR-082: 差分取得は cloudRevision と dayRefs に基づき実行します。
  • FR-082A - FR-082A: 公開HTTP APIは POST /sync/push と POST /sync/pull のみとし、CSV取り込みはローカルI/F（ProtocolImportService.importFromDirectory）で実行します。
  • FR-083 - FR-083: 競合解決はエンティティ単位LWW（updatedAt, updatedByDeviceId, mutationId 降順）で固定します。
  • FR-084 - FR-084: tombstone（削除）もLWW同一ルールで解決します。
  • FR-085 - FR-085: 競合解決は内部適用のみとし、競合件数バナーや詳細一覧は提供しません。
  • FR-086 - FR-086: 手動同期ボタンを提供し、失敗時は再試行導線を表示します。
  • FR-088 - FR-088: 同期失敗時は直近同期状態（成功/失敗）を更新し、再試行導線を表示します。
  • FR-089 - FR-089: 120秒ポーリング は実装しません。
  • FR-089A - FR-089A: session_summary.payload.exit_site_photo の更新は部分パッチ（patch_path=payload.exit_site_photo）で同期し、同一record内の他フィールドを上書きしません。

対応AT

  • AT-API-001 公開API最小化
  • AT-API-004 非暗号化キー
  • AT-SYNC-001 起動時pull復元
  • AT-SYNC-002 完了時push反映
  • AT-SYNC-003 LWW内部適用
  • AT-SYNC-004 同日同スロット競合
  • AT-SYNC-005 手動同期消し込み
  • AT-SYNC-006 復帰時失敗導線
